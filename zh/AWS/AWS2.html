☁️ AWS 雲端同步筆記（第二篇）

本頁說明如何在 VS Code 內直接將本地資料夾同步上傳至 AWS S3，無需再登入 AWS 介面。

1️⃣ 目標說明
  • 希望在 VS Code 直接「修改 → 一鍵同步 → 完成」
  • 使用 AWS CLI 指令或 VS Code Task 自動化上傳
  • 避免重複開啟 AWS Console 手動操作
  • 同步 Bucket：kao-web-static（Region: ap-northeast-1）
2️⃣ 操作路徑
本地端：C:\Users\User\Desktop\kao-aws-s3-site
雲端端：s3://kao-web-static
3️⃣ CLI 同步指令
aws s3 sync "C:\Users\User\Desktop\kao-aws-s3-site" s3://kao-web-static

這條指令可直接上傳所有有修改過的檔案。 若加上 --delete 則會清除雲端上已不存在的檔案。

4️⃣ 自動化構想
  • 在 .vscode/tasks.json 新增自動化任務「Sync to S3」
  • 按下 Ctrl + Shift + B 即可自動執行同步指令
  • 日後也可結合 GitHub Actions，自動將 push 更新同步到 S3
5️⃣ 驗證網址

同步完成後，可直接開啟以下網址查看是否更新：

http://kao-web-static.s3-website-ap-northeast-1.amazonaws.com/zh/KAO/KAO_AWS2.html

6️⃣ 補充：避免誤覆蓋的建議
  • 永遠確認同步方向：本地 → 雲端，不要反向。
  • 若只想測試差異，用 --dryrun 模式：
aws s3 sync "C:\Users\User\Desktop\kao-aws-s3-site" s3://kao-web-static --dryrun
